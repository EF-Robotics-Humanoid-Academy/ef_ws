Sensors FAQ
===========

Q: Which scripts are covered here?
A: Primary scripts in this folder:
   - `imu_plot.py`
   - `lidar_points.py`
   - `live_points.py`
   - `rgbd_cam.py`
   - `manual_streaming/*` helpers

Q: What is `imu_plot.py` for?
A: Real-time IMU stability plotting from `rt/lowstate` (roll/pitch/yaw/tilt +
   stability score), with optional odom context.

Q: How do I run `imu_plot.py`?
A: `python3 imu_plot.py --iface eth0 --domain-id 0`

Q: Useful `imu_plot.py` options?
A: `--history`, `--max-tilt-deg`, `--lowstate-topic`, `--odom-topic`,
   `--odom-type`, `--log-level`.

Q: What is `lidar_points.py` for?
A: DDS PointCloud2 top-down viewer (OpenCV) for topics like
   `rt/utlidar/cloud_livox_mid360`.

Q: How do I run `lidar_points.py`?
A: `python3 lidar_points.py --iface enp1s0 --topic rt/utlidar/cloud_livox_mid360`

Q: Useful `lidar_points.py` options?
A: `--range-m`, `--z-min`, `--z-max`, `--stride`, `--size`.

Q: What is `live_points.py` for?
A: Open3D live viewer for Livox streams (SDK wrappers), useful as a quick
   sanity check before running full SLAM.

Q: What is `rgbd_cam.py` for?
A: Laptop-side camera viewer with mode selection:
   - DDS subscribe mode,
   - VideoClient RPC mode,
   - auto fallback from DDS to RPC.

Q: How do I run `rgbd_cam.py`?
A: `python3 rgbd_cam.py --iface eth0 --mode auto`

Q: Useful `rgbd_cam.py` options?
A: `--mode {auto,dds,rpc}`, `--rgb-topic`, `--depth-topic`, `--rgb-type`,
   `--depth-type`, `--video-field`, `--rpc-timeout`, `--dds-wait`.

Q: Where are RealSense network streaming scripts?
A: `g1/scripts/sensors/manual_streaming/` includes sender/receiver and CLIP
   overlay variants (`jetson_realsense_stream.py`, `receive_realsense_gst.py`,
   `receive_realsense_gst_clip_can.py`, etc.).

Q: What if no data appears?
A: Verify domain/interface first, then verify topic/type pairing and that
   the producer script/service is running.
