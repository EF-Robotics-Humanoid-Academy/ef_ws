Safety FAQ (arm_motion/pbd/safety)
==================================

Q: What scripts are in this folder?
A: `hanger_boot_sequence.py`, `keyboard_controller.py`, `usb_controller.py`.

Q: Why is there a safety folder under PBD?
A: PBD record/replay workflows can put the robot in unusual postures. These
   utilities provide safe startup and manual repositioning before/after demos.

Q: How do I run keyboard tele-op?
A: `python3 keyboard_controller.py --iface enp1s0`

Q: Keyboard input mode options?
A: `--input pynput` (default, true key release handling) and `--input curses`.

Q: Keyboard controls?
A: `W/S` forward/back, `Q/E` strafe, `A/D` yaw, `Space` stop,
   `Z` Damp + exit, `Esc` StopMove + ZeroTorque + exit.

Q: How do I run USB gamepad tele-op?
A: `python3 usb_controller.py --iface enp1s0 --joy 0`

Q: USB mappings summary?
A: Left stick Y -> `vx`, left stick X -> `vy`, right stick X -> `vyaw`.
   Buttons: `A` Damp, `B` BalanceStand, `X` Dev/Start mode, `Y` ZeroTorque,
   `Start` StopMove.

Q: What does `hanger_boot_sequence.py` guarantee?
A: Safe startup sequence with checks around FSM/mode so operation starts from
   a stable standing state.

Q: Safety warning?
A: `ZeroTorque` removes support. Use spotter + clear workspace.
